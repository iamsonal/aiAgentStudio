    conversational and direct executions, call tools safely, work through channel-aware delivery,
    and stay observable in production.
  </p>

    <div class="stat-tile"><strong>Strategy-aware runtime</strong><br />Separate runtime strategy from interaction channel so chat, email, API, and future transports can share a common execution model.</div>
    <div class="stat-tile"><strong>Trust layers built in</strong><br />PII masking, prompt safety, human approval controls, and runtime tool gating are first-class parts of the platform.</div>
    <div class="stat-tile"><strong>Designed to extend</strong><br />Add actions, context providers, provider adapters, and memory strategies without forking the framework core.</div>

## Why Teams Use It

<CardGrid>
  <Card title="Built for Salesforce constraints" icon="seti:settings">
    The framework is designed around callout safety, user-mode data access, queueable and event-driven execution, and durable execution history.
  </Card>
  <Card title="Clear runtime control" icon="seti:graph-line">
    Strategy resolution, channel routing, trust controls, and execution lifecycle tracking are explicit rather than hidden inside prompts or UI-only workflows.
  </Card>
  <Card title="Governance by default" icon="seti:lock">
    CRUD, FLS, sharing, trust layers, and human approval controls are part of the framework model, not bolted on after the fact.
  </Card>
  <Card title="Extensible where it matters" icon="seti:code">
    Build new capabilities and provider integrations while preserving the common execution pipeline, memory model, and security posture.
  </Card>
</CardGrid>

## Core Capabilities

<CardGrid>
  <Card title="Conversational Runtime" icon="chat">
    Multi-turn agent experiences with memory, tool use, follow-up turns, and channel-aware continuity.
  </Card>
  <Card title="Direct Runtime" icon="seti:gear">
    Focused one-turn or minimal-orchestration executions for classification, enrichment, scoring, and targeted business actions.
  </Card>
  <Card title="Channel-Aware Delivery" icon="seti:mail">
    Interaction channels are handled independently from runtime strategy, allowing the framework to preserve the same execution semantics across different delivery surfaces.
  </Card>
  <Card title="Trust and observability" icon="seti:shield">
    Durable execution steps, token and cost tracking, tool reasoning capture, masking, safety checks, and HITL controls support production use.
  </Card>
</CardGrid>
