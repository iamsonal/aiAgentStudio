## What It Does

AI Agent Studio is a **production-grade orchestration system** for Salesforce-native AI experiences. It supports conversational copilots, single-purpose automation agents, email workflows, human approvals, multi-agent coordination, observability, and security controls inside Salesforce.

## Key Features

<Tabs>
  <TabItem label="Performance">

**Scalable Async Processing**: Platform Events for high-throughput production environments. Resilient processing with priority ordering, exponential backoff, and automatic recovery.

**Efficient LLM Batching**: Multiple LLM calls can be processed within a single transaction when tool execution allows it, reducing latency and job overhead.

  </TabItem>
  <TabItem label="Observability">

**Execution Storyboard**: Visual UI showing complete agent execution timeline with steps, decisions, and outcomes.

**Tool Reasoning**: LLM explains why it selected each tool, captured in execution logs for transparency and debugging.

**Token & Cost Tracking**: Per-step tracking of prompt/completion tokens and estimated costs. Build dashboards to monitor spend.

**Audit Trail**: Complete log of user inputs, LLM responses, tool calls, and results for compliance and troubleshooting.

  </TabItem>
  <TabItem label="Extensibility">

**Custom Actions**: Extend the framework with business-specific actions and integrations.

**Context Providers**: Supply dynamic business context so agents can act with better awareness of records, users, and workflows.

**LLM Providers**: Use OpenAI-compatible providers out of the box and support additional provider patterns for broader model ecosystems and enterprise deployments.

**Memory Strategies**: Tailor how the framework remembers, summarizes, and reuses prior conversation history.

  </TabItem>
</Tabs>

## Quick Start

<Aside type="tip" title="Time to First Agent">
You can have a working first agent in about 30 minutes.
</Aside>

<Steps>
1. Deploy the framework to your org.
2. Configure Named Credentials for your AI provider.
3. Create an LLM Configuration.
4. Create an AI Agent Definition.
5. Add capabilities and configure approvals.
6. Place the chat component on a Lightning page and test.
</Steps>

Start with the guided setup and iterate from observability data.
