## Security and Trust Layers

<Tabs>
  <TabItem label="Platform Security">

<ul>
  <li>Agents run in user context (no implicit privilege escalation)</li>
  <li>Object and field permissions are enforced automatically</li>
  <li>Sharing rules are respected for record-level access</li>
  <li>All execution decisions and actions are logged</li>
</ul>

  </TabItem>
  <TabItem label="Trust Layers">

<ul>
  <li>PII masking before outbound model calls</li>
  <li>Prompt safety checks for jailbreak/prompt injection patterns</li>
  <li>Configurable response modes and safety thresholds</li>
  <li>Recommended to validate in sandbox before production rollout</li>
</ul>

  </TabItem>
</Tabs>

## Quick Start

<Aside type="tip" title="Time to First Agent">
You can have a working first agent in about 30 minutes.
</Aside>

<Steps>
1. Deploy the framework to your org.
2. Configure Named Credentials for your AI provider.
3. Create an LLM Configuration.
4. Create an AI Agent Definition.
5. Add capabilities and configure approvals.
6. Place the chat component on a Lightning page and test.
</Steps>

<p><Icon name="rocket" /> Start with the guided setup and iterate from observability data.</p>
