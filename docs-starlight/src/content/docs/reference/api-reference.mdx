# API Reference

The core framework exposes REST surfaces for agent execution and normalized inbound traffic.

## Agent REST Surface

Base path:

```text
/services/apexrest/ai/agent
```

Common endpoints:

- `POST /process`
- `POST /hitl/execute`
- `POST /hitl/followup`
- `POST /resume`

### `/process`

Use this endpoint to start a new execution or continue an existing one in the agent REST flow.

Important request notes:

- `originalUserId`, `agentDefinitionId`, and `turnIdentifier` are required
- `executionId` is optional when continuing an existing execution
- `interactionSessionId` is optional when you want durable session continuity across executions
- `userMessage` may be omitted for non-interactive or non-chat scenarios

## Normalized Ingress Surface

Base path:

```text
/services/apexrest/ai/ingress
```

This REST surface is intended for middleware or channel integrations that already speak the
framework's normalized inbound contract.

<Aside type="note" title="Why there are two surfaces">
The `/ai/agent` endpoints are centered on agent execution. The `/ai/ingress` surface exists for
channel-style ingress flows that need normalization, routing, and session resolution before the
agent execution begins.
</Aside>

## Authentication

Use normal Salesforce authentication such as a session token or OAuth bearer token.

```text
Authorization: Bearer YOUR_ACCESS_TOKEN
Content-Type: application/json
```
