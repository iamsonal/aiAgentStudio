# API Reference

## Base Endpoint

`/services/apexrest/ai/agent`

## Common Endpoints

- `POST /process` for chat turn processing
- `POST /hitl/execute` for executing an approved HITL action
- `POST /hitl/followup` for queueing follow-up LLM work after rejection, expiry, or decline
- `POST /resume` for resuming an execution in service-user context

## Authentication

Use Salesforce authentication (session token or OAuth) and include:

<Tabs>
  <TabItem label="Headers">

```text
Authorization: Bearer YOUR_ACCESS_TOKEN
Content-Type: application/json
```

  </TabItem>
  <TabItem label="cURL Example">

```bash
curl -X POST "https://YOUR-INSTANCE.salesforce.com/services/apexrest/ai/agent/process" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"originalUserId":"005...","agentDefinitionId":"a0Y...","turnIdentifier":"turn-001","userMessage":"hello"}'
```

  </TabItem>
</Tabs>

## Request Notes

- `sessionId` is optional on `/process`; omit it to start a new execution.
- `originalUserId`, `agentDefinitionId`, and `turnIdentifier` are required on `/process`.
- `/hitl/execute` expects the pending action, execution, capability, and serialized tool arguments produced by the HITL workflow.
- `/resume` is used by service-user routing and recovery flows, not normal end-user chat traffic.
